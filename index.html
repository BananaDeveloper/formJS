Intro

This module lets you validate forms.
You can specify form/field options to:

  • skip feedback on UI
  • specify CSS class(es) for dirty/error/valid fields
  • specify which events will fire the validation
  • manage AJAX calls: handle JSON data, change settings and set callbacks
  • and more...

Browsers Support

formJS is compatible with ( desktop & mobile versions ):

  • Chrome
  • Firefox
  • Internet Explorer 10+ / Edge *
  • Safari

* for compatibility with IE 10/11 and old versions of Edge you have to load some JS polyfills ( from polyfill.io or else ):

HTML

This is the basic HTML structure to make the validation works ( Bootstrap CSS classes are optional ).
Some notations about form attributes:

  • novalidate ( required ) enables JS field validation
  • data-ajax-settings ( optional ) to specify new options for the AJAX call as JSON object inside the HTML attribute. See AJAX Options for details.
  • data-formjs-init ( optional ) if set to "false", the plugin will not initialize the events listeners but you can use its methods and so on ( for example if you just want to use getFormJSON method ). See non-fully-initialized forms

And some notations about field attributes:

  • data-formjs-question ( optional empty attribute ) used for visual feedback on valid/invalid fields and/or character count functionality ( to be used on a container element for the field )
  • required ( required ) for fields that require an answer
  • type ( required )
  • data-subtype ( optional ) for special field types ( eg: VAT number, fiscal code, etc - see the full list )
  • data-exclude-json ( optional empty attribute ) used to avoid the plugin to take in consideration the field when building the JSON ( used inside getFormJSON method )

Below, the HTML example: 

<form action="/mypath/mypage.php" method="POST" name="my-form-example" novalidate>

    <!-- BASIC REQUIRED TEXT INPUT -->
    <div data-formjs-question="">
        <div class="answers-box form-group">
            <input name="myName" type="text" class="form-control" required />
            <label>Name</label>
        </div>
        <div class="field-error-message small">Write your name!</div>
    </div>
    
    <!-- REQUIRED FIELD WITH HTML STANDARD TYPE ( EMAIL ) -->
    <div data-formjs-question="" class="form-group">
        <div class="answers-box">
            <input name="myEmail" type="email" class="form-control" required />
            <label>Email</label>
        </div>
        <div class="field-error-message small">Check your email address, it is not valid!</div>
    </div>
    
    <!-- REQUIRED FIELD WITH SPECIAL TYPE:
         BASIC TYPE IS text
         SPECIAL TYPE ( NAMED SUBTYPE ) IS INSIDE data-subtype ATTRIBUTE -->
    <div data-formjs-question="" class="form-group">
        <div class="answers-box">
            <input name="myMobileNumber" type="text" data-subtype="mobile-number" class="form-control" required />
            <label>Mobile number</label>
        </div>
        <div class="field-error-message small">Check your mobile number, it is not valid!</div>
    </div>
    
    <!-- FIELD WITH SPECIAL TYPE AND VALIDATED ONLY IF FILLED:
         BASIC TYPE IS text
         SPECIAL TYPE ( NAMED SUBTYPE ) IS INSIDE data-subtype ATTRIBUTE
         FIELD VALUE IS EVALUATED ONLY IF FILLED ( SEE data-validate-if-filled ATTRIBUTE ) -->
    <div data-formjs-question="" class="form-group">
        <div class="answers-box">
            <input name="websiteUrl" type="text" data-subtype="url" class="form-control" data-validate-if-filled="" />
            <label>Website</label>
        </div>
        <div class="field-error-message small">Check your website url, it is not valid!</div>
    </div>
    
    <!-- BASIC TEXT INPUT NOT MANDATORY -->
    <div data-formjs-question="">
        <div class="answers-box form-group">
            <input name="myInfo" type="text" class="form-control" />
            <label>Info</label>
        </div>
    </div>
    
    <button type="submit" class="btn btn-default">SEND</button>
    
</form>

Initialization

Here an example of initialization: 

var formEl = document.querySelector('form');
// or with CSS string selector:
// var formEl = '.my-container .my-form';

var options = {};

var formInstance = new Form( formEl, options );
  • formEl: ( required ) must be a single form node ( not a nodelist! ) or a CSS string selector
  • options: ( optional ) a JS object with fieldOptions and formOptions ( see Options section )

You will read about "non-fully-initialized forms". It means that the form has data-formjs-init="false" attribute, so the plugin will not add the event listeners to validate the fields and to submit the form.

IMPORTANT: If Form namespace is not available, you can use FormJS namespace.

Options:

formJS takes an object as second argument ( options ), as follow: 

{
    fieldOptions:   {...},
    formOptions:    {...}
}

function

Fires when the paste event has been prevented on the field(s) ( see "preventPasteFields" option ).

onValidation

function

Fires just after a field has been validated.

preventPasteFields

string

CSS string selector for which the paste command will be disabled

skipUIfeedback

boolean

if true, the "has-error/has-error-switch" class will not be added ( for UI feedback as said by the name )

strictHtmlValidation

boolean

activate JS controls for strict JS validation ( as for HTML5 attributes, eg: mexlength that does not work on Android Chrome/stock borwsers )

string

events on which the validation will fire ( only for required fields )
( not used for non-fully-initialized forms )

    checkDirtyField:        false,
    cssClasses: {
        dirty:              'dirty-field',
        error:              'has-error',
        valid:              'is-valid'

    maxFileSize:            10,
    onPastePrevented:       null,
    onValidation:           null,

    skipUIfeedback:         false,
    strictHtmlValidation:   true,

AJAX Options

Below the default AJAX Options used for AJAX form submit ( can be changed via data-ajax-settings attribute ): 

{
    async:          true,
    cache:          false,
    contentType:    'application/x-www-form-urlencoded; charset=UTF-8',
    data:           formDataJSON,
    headers:        {},
    method:         'POST',
    timeout:        0,
    url:            location.href
}

Below all the AJAX options that can be changed:

  • async: if true will make an async ajax call, if false will make a sync ajax call.
  • cache: ( only for GET ajax calls ) if true will add a timestamp parameter to the url query string.
  • contentType: content type of the ajax request ( anything put inside the enctype html form attribute ).
  • headers: object of key/value pairs to set the request headers.
  • mimeType: a mime type to override the XHR mime type.
  • timeout: ( in milliseconds ) in greater than 0, after the specified time, the ajax call will be aborted if still pending.
  • xhrFields: object of fieldName-fieldValue pairs to set on the native XHR object.
  • data ( formDataJSON ) is automatically generated by the plugin and can be changed via beforeSend callback function
  • method is by default 'POST' but it takes the "method" html form attribute if set
  • url is by default the page url but it takes the "action" html form attribute if set

Below, an example of AJAX settings passed via "data-ajax-settings" ( they will override the same options specified via HTML attributes ): 

<form action="/mypath/mypage.php"
        method="POST"
        name="my-form-example"
        novalidate
        data-ajax-settings='{
                                "dataType": "json",
                                "myoption": [1,2,3],
                                "option3": {
                                "intOpt": "hello"
                                }
                            }'
> ... </form>

Methods

name
description & parameters
return value 
destroy()

Remove all events listeners from the form instance. 

getFormJSON()

Get a JSON object of the form.

a JSON object with pairs of field-name / field-value like: 

{
    name:   'Valerio',
    city:   'Milan',
    website:'http://www.valeriodipunzio.com'
}
isValidField( field[, fieldOptions] )

Vaidate the field based on the field options of the instance or the ones passed as second argument.

  • field: CSS string selector ( or node element * ) for the form field ( input, select, textarea )
  • fieldOptions: ( Optional ) JS object. For default values see Field Options section

* best to use a CSS string selector so that it looks for the field inside the instance form element.
Example:
 

formInstance.isValidField('[name="myEmail"]');

Boolean 

isValidForm( [options] )

Vaidate the form based on the form options of the instance or the ones passed as argument.

  • options: ( Optional ) JS object with fieldOptions and/or formOptions. For default values see Options section

Object like: 

{
    fields: [{
        field:  element,
        result: true
    }],
    result:     true
}
  • fields: ( array of JS objects ) a list of the form fields with field ( JS node element ) and result ( boolean )
  • result: ( boolean ) check for the form, if it is valid or not 
submit( [options, [event]] )

To manually trigger the form submit.

  • options: ( Optional ) JS object. Can contain fieldOptions and formOptions. For default values see Options section
  • event: ( Optional - Required for submit of non-fully-initialized forms ) JS event object for submit event.

It will:

  • validate the form according to the fieldOptions and formOptions ( passed inside options param )
  • ( for AJAX forms ) create the JSON data to send via AJAX call as for getFormJSON method
  • ( for AJAX forms ) make the AJAX call to the url specified as action of the HTML form
  • ( for AJAX forms ) trigger callback functions as defined in options param

Prototype Methods

You can use these methods ONLY from the original Form object ( not from a new instance - examples below ):

name
parameters
notes 
addValidationRules( newRules )
  • newRules: JS object containing the new validation rule(s) to add:
    {
    zip: function( zipCode ){ return /[5-9]{3}$/.test( zipCode ); }
}

The key / value pairs must be passed as:

  • key: the same used as data-subtype ( or type ) attribute, without dash and with Capital letter instead.
    Eg: to validate "mobile-number" ( as data-subtype ) the rule is "mobileNumber"
  • value: a function that must return a boolean
This will affect both old and new instances.
Can also be used to override existing validation rules. 
setOptions( options )
  • options: JS object with the fieldOptions/formOptions to override ( see Options section )
This will affect ONLY new instances

Examples of usage: 

Form.addValidationRules({
    zip: function( string ){ return /^[5-9]{3}$/.test( string ); }
});

Form.setOptions( { fieldOptions: {maxFileSize: 100} } );

Set a counter for the field ( for example for a message in a textarea ).

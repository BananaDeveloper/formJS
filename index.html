Usage

This module lets you validate forms.
You can specify form/field options to:

  • skip feedback on UI
  • specify CSS class(es) for dirty/error/valid fields
  • check if a field is "dirty"
  • specify which events will fire the validation
  • handle JSON data generated from a form
  • change AJAX call settings
  • callbacks for AJAX call

Dependencies

formJS is developed with these dependencies:

  • jQuery 1.12.4 
<!-- JQUERY -->
<script src="js/vendors/jquery-1.12.4.min.js"></script>

The demo is developed with Bootstrap CSS 3.3.7

HTML

This is the basic HTML structure to make the validation works ( some Bootstrap classes are optional ).
Some notations about form/fields attributes:

  • novalidate: ( mandatory ) for the form ( fields will be validated via JS )
  • data-ajax-submit: ( optional ) for the form. To let the plugin manage the form with an AJAX call.
  • data-ajax-settings: ( optional ) for the form, to specify new options for the AJAX call as JSON object inside the HTML attribute. Used together with data-ajax-submit attribute.
    See below for details.
  • required: ( mandatory ) for fields that require an answer
  • type: ( mandatory )
  • data-subtype: ( optional ) for special field types ( eg: VAT number, fiscal code, etc - see the full list )
  • data-exclude-json: ( optional ) for fields, used to avoid the plugin to take in consideration the field when building the JSON ( used inside getFormJSON method, so inside submitAjaxForm method )
  • data-formjs-question: ( optional ) used for visual feedback on valid/invalid fields and/or character count functionality

For other validations see the full list 

<form action="/mypath/mypage.php" method="POST" name="my-form-example" novalidate>

    <!-- BASIC REQUIRED TEXT INPUT -->
    <div data-formjs-question="">
        <div class="answers-box form-group">
            <input name="myName" type="text" class="form-control" required />
            <label>Name</label>
        </div>
        <div class="field-error-message small">Write your name!</div>
    </div>
    
    <!-- REQUIRED FIELD WITH HTML STANDARD TYPE ( EMAIL ) -->
    <div data-formjs-question="" class="form-group">
        <div class="answers-box">
            <input name="myEmail" type="email" class="form-control" required />
            <label>Email</label>
        </div>
        <div class="field-error-message small">Check your email address, it is not valid!</div>
    </div>
    
    <!-- REQUIRED FIELD WITH SPECIAL TYPE:
         BASIC TYPE IS text
         SPECIAL TYPE ( NAMED SUBTYPE ) IS INSIDE data-subtype ATTRIBUTE -->
    <div data-formjs-question="" class="form-group">
        <div class="answers-box">
            <input name="myMobileNumber" type="text" data-subtype="mobile-number" class="form-control" required />
            <label>Mobile number</label>
        </div>
        <div class="field-error-message small">Check your mobile number, it is not valid!</div>
    </div>
    
    <!-- FIELD WITH SPECIAL TYPE AND VALIDATED ONLY IF FILLED:
         BASIC TYPE IS text
         SPECIAL TYPE ( NAMED SUBTYPE ) IS INSIDE data-subtype ATTRIBUTE
         FIELD VALUE IS EVALUATED ONLY IF FILLED ( SEE data-validate-if-filled ATTRIBUTE ) -->
    <div data-formjs-question="" class="form-group">
        <div class="answers-box">
            <input name="websiteUrl" type="text" data-subtype="url" class="form-control" data-validate-if-filled="" />
            <label>Website</label>
        </div>
        <div class="field-error-message small">Check your website url, it is not valid!</div>
    </div>
    
    <!-- BASIC TEXT INPUT NOT MANDATORY -->
    <div data-formjs-question="">
        <div class="answers-box form-group">
            <input name="myInfo" type="text" class="form-control" />
            <label>Info</label>
        </div>
    </div>
    
    <button type="submit" class="btn btn-default">SEND</button>
    
</form>

AJAX Options

Below the default AJAX Options used inside submitAjaxForm method ( can be changed via data-ajax-settings attribute ): 

{
    cache: false,
    data: formData,
    url: $form.attr('action'),
    method : $form.attr('method') || 'POST'
}

formData is automatically generated by the plugin and can be changed via beforeSend callback function

Below, an example of AJAX settings passed via HTML data attribute: 

<form action="/mypath/mypage.php"
         method="POST"
         name="my-form-example"
         novalidate
         data-ajax-settings='{
                                 "dataType": "json",
                                 "myoption": [1,2,3],
                                 "option3": {
                                    "intOpt": "hello"
                                 }
                             }'
> ... </form>

Initialization

You can initialize every form inside the page to be managed by formJS with very basic JS code: 

FORM.init();

Options:

formJS has "options" as second argument, as follow: 

{
    fieldOptions: {...},
    formOptions: {...}
}

Field Options

type
description

boolean

wether or not to use the checkDirtyField() method when validating a field

cssClasses

JS object as follow:

  • dirty: ( string ) CSS class(es) to be applied if the field "dirty" ( has some value )
  • error: ( string ) CSS class(es) to be applied if the field is not valid
  • errorMultiChoice: ( string ) CSS class(es) to be applied if the field with multiple selectable answers ( eg: some checboxes ) is not valid
  • valid: ( string ) CSS class(es) to be applied if the field is valid 
    {
        dirty: 'dirty-field',
        error: 'has-error',
        errorMultiChoice: 'has-error-switch',
        valid: 'is-valid'
    }

focusOnRelated

boolean

wether or not to focus on the related (input/select) field when selecting its radio answer

onValidation

function

used ONLY inside init() and submitAjaxForm() methods. Fires just after a field has been validated.

preventPasteFields

string

CSS string selector for which the paste command will be disabled

skipUIfeedback

boolean

if true, the "has-error/has-error-switch" class will not be added ( for UI feedback as said by the name )

strictHtmlValidation

boolean

activate JS controls for strict validation ( as for HTML5 attributes, eg: mexlength that does not allow to type chars that exceed its value )

validateOnEvents

string

events on which the validation will fire ( only for required fields ) - used ONLY inside init() method

Default field options are: 

    {
        checkDirtyField: false,
        cssClasses: {
            dirty: 'dirty-field',
            error: 'has-error',
            errorMultiChoice: 'has-error-switch',
            valid: 'is-valid'
        },
        focusOnRelated: true,
        onValidation: null,
        preventPasteFields: '[type="password"], [data-equal-to]',
        skipUIfeedback: false,
        strictHtmlValidation: false,
        validateOnEvents: 'input change'
    }

Methods

formJS has these useful methods:

name
parameters
return value 
addValidationRules( newRules )
  • newRules: JS object containing the new validation rule(s) to add:
    {
    zip: function( zipCode ){ return /[5-9]{3}$/.test( zipCode ); }
}

The key / value pairs must be passed as:

  • key: the same used as data-subtype ( or type ) attribute, without dash and with Capital letter instead.
    Eg: to validate "mobile-number" ( as data-subtype ) the rule is "mobileNumber"
  • value: a function that must return a boolean 
checkDirtyField( $fields )
  • $fields: jQuery collection of fields ( or a single field ). To know if a field has a value or is empty ( useful in case, for example, where you want to move the label of a field - see all examples above but Basic UI ). 
getFormJSON( $form )
  • $form: jQuery object for the form you want to get the JSON from.

a JSON object with pairs of field-name / field-value like: 

{
    name: 'Valerio',
    city: 'Milan',
    website: 'http://www.valeriodipunzio.com'
}
isValidField( $field[, fieldOptions] )
  • $field: jQuery object for the form field ( input, select, textarea )
  • fieldOptions: ( Optional ) JS object. For default values see Options section

Boolean 

isValidForm( $form[, options] )
  • $form: jQuery object for the form
  • options: ( Optional ) JS object. Can contain fieldOptions and formOptions. For default values see Options section

Object like: 

{
    $fields: [{
        $field: $('.my-field')
        result: true
    }],
    result: true
}
  • $fields: ( array of JS objects ) a list of the form fields with $field ( jQuery object ) and result ( boolean )
  • result: ( boolean ) check for the form, if it is valid or not 
init( [$form[, options]] )
  • $form: ( Optional ) jQuery object for the form(s)
  • options: ( Optional ) JS object. Can contain fieldOptions and formOptions. For default values see Options section

The validation is executed on every field with required attribute on events as defined in fieldOptions. For default values see Options section

This method will validate specific forms ( if passed ) or every form ( if a falsy value is passed ) that:

  • has novalidate attribute
  • is not a survey form ( with class "survey-form" )

If the form has "data-ajax-submit" attribute, submitAjaxForm method will be used on form submit. 

submitAjaxForm( $form[, options] )
  • $form: jQuery object for the form
  • options: ( Optional ) JS object. Can contain fieldOptions and formOptions. For default values see Options section

It will:

  • validate the form according to the fieldOptions and formOptions ( passed inside options param )
  • create the JSON data to send via AJAX call as for getFormJSON method
  • make the AJAX call to the url specified as action of the HTML form
  • trigger callback functions as defined in options param

Validation subtypes

Special fields, with data-subtype attribute, will be validated with rules as follow ( you can add and/or override these rules with addValidationRules method ):

data-subtype value
validation rule
cap

/[0-9]{5}$/


VALID ITALIAN CAP WITH 5 DIGITS

date

/^(0?[1-9]|[12][0-9]|3[01])([ \/\-.])(0?[1-9]|1[012])\2([0-9][0-9][0-9][0-9])(([ -])([0-1]?[0-9]|2[0-3]):[0-5]?[0-9]:[0-5]?[0-9])?$/


DATE AS ITALIAN SYNTAX WITH / WITHOUT TIME
dd mm yyyy | dd/mm/yyyy | dd.mm.yyyy | dd-mm-yyyy | dd/mm/yyyy-hh:mm:ss ( WITH SPACE / . - AS SEPARATOR FOR THE DATE )

email

/(\w+)\@(\w+)\.[a-zA-Z]/

EMAIL MUST BE AT LEAST ( FOR EXAMPLE ): a@a.a

fiscal-code

/^(?:[B-DF-HJ-NP-TV-Z](?:[AEIOU]{2}|[AEIOU]X)|[AEIOU]{2}X|[B-DF-HJ-NP-TV-Z]{2}[A-Z]){2}[\dLMNP-V]{2}(?:[A-EHLMPR-T](?:[04LQ][1-9MNP-V]|[1256LMRS][\dLMNP-V])|[DHPS][37PT][0L]|[ACELMRT][37PT][01LM])(?:[A-MZ][1-9MNP-V][\dLMNP-V]{2}|[A-M][0L](?:[\dLMNP-V][1-9MNP-V]|[1-9MNP-V][0L]))[A-Z]$/i


read this post

hex-color

/^#?([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$/


HEX COLOR WITH/WITHOUT #.
LOWERCASE AND/OR UPPERCASE
CAN BE 3 OR 6 CHARACTERS ( 0fa | 00FFaa )

landline-number

/^((00|\+)\d{2}[\-\. ]??)??(((0[\d]{1,4}))([\/\-\. ]){0,1}([\d, ]{5,10}))$/


LANDLINE PREFIX:
+39 | 0039 | not-set ( ALSO WITH INTERNATIONAL PREFIXES WITH 2 DIGITS )

LANDLINE NUMBER MUST START WITH 0 ( AS FOR ITALIAN ONES ):
1234567890 | 12 34567890 | 123456789 | 1234 56789 ( ALSO WITH . - / AS SEPARATOR )

mobile-number

/^((00|\+)\d{2}[\-\. ]??)??3\d{2}[\-\. ]??\d{6,7}$/


+39 | 0039 | not-set ( ALSO WITH ALL INTERNATIONAL PREFIXES WITH 2 DIGITS )
MOBILE NUMBER MUST START WITH 3
3234567890 | 323 4567890 ( ALSO WITH . OR - AS SEPARATOR )

number

/[+-]?([0-9]*[.])?[0-9]+/


CHECK IF IT IS A NUMBER ( BOTH INTGER AND FLOATING ARE VALID )

numberFloat

/[+-]?([0-9]*[.])[0-9]+/


CHECK IF IT IS A FLOATING NUMBER
VALID NUMBERS: 123.456 | .123

numberInteger

/^\d+$/


CHECK IF IT IS AN INTEGER NUMBER

password

/^(?=.*\d)(?=.*[a-z])(?=.*[A-Z])[0-9a-zA-Z]{8,}$/


PASSWORD ( NO SPECIAL CHARACTERS ) WITH AT LEAST:
ONE DIGIT and ONE LOWERCASE and ONE UPPERCASE and MIN LENGTH OF 8 CHARACTERS

url

/^((https?|ftp|file):\/\/)?([\da-z\.-]+)\.([a-z\.]{2,6})([\/\w \.-]*)*\/?$/


MUST NOT CONTAIN PARAMETERS:
www.mysite.com/index.html --> VALID URL
www.mysite.com/index.html?v=hello --> INVALID URL

username

/^[a-zA-Z0-9_-]{3,24}$/


USERNAME WITH LETTERS/NUMBERS AND - OR _ WITH MIN LENGTH 3 AND MAX LENGTH 24

vat-number

/^(IT){0,1}[0-9]{11}$/i


VAT NUMBER CAN CONTAIN OR NOT THE 'IT' STRING ( AS FOR ITALIAN VAT NUMBERS ) AND THEN 11 NUMBERS

Other validations

Other types of validations can be enabled with HTML data-* attributes:

HTML attribute
value
data-validate-if-filled

No value needed.
The field will be validated only if filled.
If empty, the field will be evaluated as valid.

data-required-from
( use together with data-require-more )

The "id" attribute of the field with "data-require-more" attribute


Used to bind the answer with "data-required-from" attribute ( eg: a text answer ) to be mandatory when the answer with "data-require-more" attribute is selected ( es: a radio answer )

data-equal-to

The "name" attribute of the field this answer value must be equal to


Eg: for a confirm email/password field

data-exact-length

The exact number of characters the answer must have ( no more nor less )


Eg: a field that must have a specific number of characters, such as for cents ( 2 digits )

Other utilities

Other utilities for fields validation activated by setting a data-* attribute on the field:

HTML attribute
description
data-char-count
( use together with data-char-length )
( optionally with maxlength and data-char-maxlength )

setup a counter for the field ( for example for a message in a textarea ).
Related HTML attributes:

  • data-char-length: used on a tag where you want to print the number of characters typed inside the field.
  • maxlength: used on the field to set the maxlength ( can be validated via JS by setting strictHtmlValidation to true ). the value will be printed as text inside the tag with data-char-maxlength attribute.
  • data-char-maxlength: used on a tag where you want to print the value of maxlength of the field.

Intro

This module lets you validate forms.
You can specify form/field options to:

  • skip feedback on UI
  • specify CSS class(es) for dirty/error/valid fields
  • specify which events will fire the validation
  • manage AJAX calls: handle JSON data, change settings and set callbacks
  • and more...

Dependencies

No dependencies.
This demo is developed with Bootstrap CSS but it is not a dependency.

Include the formJS script in your page: 

<!-- FORM JS Standard ( using fetch API for AJAX call ) -->
<script src="dist/formJS.min.js"></script>

OR

<!-- FORM JS XHR ( using XMLHttpRequest API for AJAX call ) -->
<script src="dist/formJS.xhr.min.js"></script>

HTML

This is the basic HTML structure to make the validation works ( Bootstrap CSS classes are optional ).
Some notes about form attributes:

  • novalidate ( required ) enables JS field validation

And some notes about field attributes:

  • data-formjs-question ( optional | empty attribute ) used for visual feedback on valid/invalid/dirty fields and/or character count functionality ( to be used on a container element for the field )
  • required ( required ) for fields that require an answer.
  • name ( required ) used inside "getFormData" method to collect form data.
  • type ( required ) used for field validation.
  • data-type ( optional ) by now only "number" is supported. See "strictHtmlValidation" option in Field Options section for details.
  • data-subtype ( optional ) for special field types ( eg: VAT number, fiscal code, etc - see the full list ).
  • data-exclude-data ( optional | empty attribute ) used to avoid the plugin to take in consideration the field when collecting form data ( used inside "getFormData" method ).

Below, the HTML example: 

<form action="/mypath/mypage.php" method="POST" name="my-form-example" novalidate>

    <!-- BASIC REQUIRED TEXT INPUT -->
    <div data-formjs-question="">
        <div class="answers-box form-group">
            <input name="myName" type="text" class="form-control" required />
            <label>Name</label>
        </div>
        <div class="field-error-message small">Write your name!</div>
    </div>
    
    <!-- REQUIRED FIELD WITH HTML STANDARD TYPE ( EMAIL ) -->
    <div data-formjs-question="" class="form-group">
        <div class="answers-box">
            <input name="myEmail" type="email" class="form-control" required />
            <label>Email</label>
        </div>
        <div class="field-error-message small">Check your email address, it is not valid!</div>
    </div>
    
    <!-- REQUIRED FIELD WITH SPECIAL TYPE:
         BASIC TYPE IS text
         SPECIAL TYPE ( NAMED SUBTYPE ) IS INSIDE data-subtype ATTRIBUTE -->
    <div data-formjs-question="" class="form-group">
        <div class="answers-box">
            <input name="myMobileNumber" type="text" data-subtype="mobile-number" class="form-control" required />
            <label>Mobile number</label>
        </div>
        <div class="field-error-message small">Check your mobile number, it is not valid!</div>
    </div>
    
    <!-- FIELD WITH SPECIAL TYPE AND VALIDATED ONLY IF FILLED:
         BASIC TYPE IS text
         SPECIAL TYPE ( NAMED SUBTYPE ) IS INSIDE data-subtype ATTRIBUTE
         FIELD VALUE IS EVALUATED ONLY IF FILLED ( SEE data-validate-if-filled ATTRIBUTE ) -->
    <div data-formjs-question="" class="form-group">
        <div class="answers-box">
            <input name="websiteUrl" type="text" data-subtype="url" class="form-control" data-validate-if-filled="" />
            <label>Website</label>
        </div>
        <div class="field-error-message small">Check your website url, it is not valid!</div>
    </div>
    
    <!-- BASIC TEXT INPUT NOT MANDATORY -->
    <div data-formjs-question="">
        <div class="answers-box form-group">
            <input name="myInfo" type="text" class="form-control" />
            <label>Info</label>
        </div>
    </div>
    
    <button type="submit" class="btn btn-default">SEND</button>
    
</form>

Initialization

Here an example of initialization: 

var formEl = document.querySelector('form');

// OR WITH CSS STRING SELECTOR:
// var formEl = '.my-container .my-form';

var options = {};

var formInstance = new Form( formEl, options ).init();

// IN SOME CASES YOU COULD NEED TO INITIALIZE THE FORM IN THIS WAY:
// var formInstance = new Form( formEl, options );
// formInstance.init();
  • formEl: ( required ) must be a single form node ( not a nodelist! ) or a CSS string selector
  • options: ( optional ) a JS object with fieldOptions and formOptions ( see Options section )

IMPORTANT: If Form namespace is not available, you can use FormJS namespace.

Options:

formJS takes an object as second argument ( options ), as follow: 

{
    fieldOptions:   {...},
    formOptions:    {...}
}

Field Options

name
type
description

checkDirtyField

boolean

Wether or not to check if the field is dirty ( has some value ) or not.
If set to true, the "dirty CSS class" will be applied to the container element with data-formjs-question attribute - or the field itself, if not.

cssClasses

object

JS object with keys:

  • dirty: ( string ) CSS class(es) to be applied if the field "dirty" ( has some value )
  • error: ( string ) CSS class(es) to be applied if the field is not valid ( in any error case )
  • errorEmpty: ( string ) additional CSS class(es) to be applied if the field value is empty
  • errorRule: ( string ) additional CSS class(es) to be applied if the field value does not comply the validation rule
  • valid: ( string ) CSS class(es) to be applied if the field is valid

These classes are useful to show different error messages based on the error type and/or change the UI.

focusOnRelated

boolean

Wether or not to focus on the related (input/select) field when selecting its radio answer

handleFileUpload

boolean

Wether or not the plugin must handle the file upload. It is a simple file upload via ajax.
You must also set the enctype="multipart/form-data" attribute to the form.
( NOT used if ajaxSubmit: false )

handleValidation

boolean

Wether or not the plugin must handle the fields validation ( as single check and on form submit ).

maxFileSize

number

Size limit ( in MB ) for each file to upload.
Set to 0 ( zero ) if you want to disable the size check.
Inside the field error message container you can put a span (or else) with data-max-file-size attribute to print the size limit

onPastePrevented

function / array of functions

Fires when the paste event has been prevented on the field(s) ( see "preventPasteFields" option ).
( NOT fired if handleValidation: false )

Additionally, the "this" keyword is bound to the Form instance.

Arguments

  • fieldEl: JS node element

onValidation

function / array of functions

Fires just after a field has been validated.

Additionally, the "this" keyword is bound to the Form instance.

Arguments

  • fieldsList: ( Array ) list of JS objects: 
    [{
    fieldEl: fieldEl,
    result: true,
    errors: {}
}]
    • field: JS node element
    • result: boolean
    • errors: JS object ( only in case of invalid field )

preventPasteFields

string

CSS string selector for which the paste command will be disabled

skipUIfeedback

boolean

If true, the "has-error/has-error-switch" class will not be added ( for UI feedback as said by the name )

strictHtmlValidation

boolean

Activate JS controls for strict JS validation ( as for HTML5 attributes or emulation )

maxlength
problem:
it does not work on Android Chrome/stock borwsers
solution:
emulation of working maxlength attribute: avoid to write more than specified length.
data-type="number"
problem:
in some browsers, inputs with type="number" allow the user to insert non-numeric chars.
solution:
emulation of working type="number" attribute thanks to data-type="number" attribute: allow to write only numeric values with/without "+", "-", ".", ",".

validateOnEvents

string

Events on which the validation will fire ( only for required fields )
( NOT used if handleValidation: false )

Default field options: 

{
    checkDirtyField:        false,
    cssClasses: {
        dirty:              'is-dirty',
        error:              'has-error',
        errorEmpty:         'has-error-empty',
        errorRule:          'has-error-rule',
        valid:              'is-valid'
    },
    focusOnRelated:         true,
    handleFileUpload:       true,
    handleValidation:       true,
    maxFileSize:            10,
    onPastePrevented:       null,
    onValidation:           null,
    preventPasteFields:     '[type="password"], [data-equal-to]',
    skipUIfeedback:         false,
    strictHtmlValidation:   true,
    validateOnEvents:       'input change'
}

AJAX Options

Below an explanation of AJAX options ( set these via HTML attributes, where possible, will override the ones from JS options ):

for formJS with fetch API:

  • See MDN documentation

for formJS with XMLHttpRequest API:

  • async: if true will make an async ajax call, if false will make a sync ajax call.
  • cache: ( only for GET ajax calls ) if true will add a timestamp parameter to the url query string.
  • contentType: content type of the ajax request ( or as "enctype" html form attribute if set ).
  • headers: object of key/value pairs to set the request headers.
  • method: it can also take the "method" html form attribute if set
  • mimeType: a mime type to override the XHR mime type.
  • timeout: ( in milliseconds ) in greater than 0, after the specified time, the ajax call will be aborted if still pending.
  • url: by default is the current page url but it can also take the "action" html form attribute if set
  • xhrFields: object of fieldName-fieldValue pairs to set on the native XHR object.

Get a JS object of the form with all the answers.

Run a first validation for fields that already have a value set.
Useful to call after constructor function but not mandatory.

The Form instance 
isValidField( field[, fieldOptions] )

Validate the field based on the field options of the instance or the ones passed as second argument.

  • field: CSS string selector ( or node element * ) for the form field ( input, select, textarea )
  • fieldOptions: ( Optional ) JS object. For default values see Field Options section

* best to use a CSS string selector so that it looks for the field inside the instance form element.
Example:
 

formInstance.isValidField('[name="myEmail"]');

Object like: 

{
    fieldEl: element,
    result:  false,
    errors:  {...}
}
  • fieldEl: JS node element
  • result: boolean
  • errors: ( JS object ) only in case of invalid field 
isValidForm( [options] )

Validate the form based on the form options of the instance or the ones passed as argument.

  • options: ( Optional ) JS object with fieldOptions and/or formOptions. For default values see Options section

Object like: 

{
    fields: [{
        fieldEl:  element,
        result:   false,
        errors:   {...}
    }],
    result:       false
}
  • fields: ( array of JS objects ) a list of the form fields as returned by "isValidField" method
  • result: ( boolean ) check for the form, if it is valid or not 
submit( [options[, event]] )

To manually trigger the form submit.

  • options: ( Optional ) JS object. Can contain fieldOptions and formOptions. For default values see Options section
  • event: ( Optional | Required if handleSubmit: false ) JS event object for submit event.

It will:

  • validate the form according to the fieldOptions and formOptions ( passed inside options param )
  • ( for AJAX forms ) create the JSON data to send via AJAX call as for getFormData method
  • ( for AJAX forms ) make the AJAX call to the url specified as action of the HTML form
  • ( for AJAX forms ) trigger callback functions as defined in options param 

validateField( field[, fieldOptions] )

Fires validation for the specified field ( with "isValidField" method ).
Then, this also fires feedback on UI ( basic feedback with css classes ) and callback function(s) ( "onValidation" in "fieldOptions" ).

  • field: CSS string selector ( or node element * ) for the form field ( input, select, textarea )
  • fieldOptions: ( Optional ) JS object. For default values see Field Options section

* best to use a CSS string selector so that it looks for the field inside the instance form element.
Example:
 

formInstance.validateField('[name="myEmail"]');

as for "isValidField" method 

validateForm( [options] )

Fires validation for the form ( with "isValidForm" method ).
Then, this also fires feedback on UI for each field ( basic feedback with css classes ) and callback function(s) passing the full array of fields ( "onValidation" in "fieldOptions" ).

  • options: ( Optional ) JS object with fieldOptions and/or formOptions. For default values see Options section

as for "isValidForm" method

Prototype Methods

You can use these methods ONLY from the original Form object ( not from a new instance - examples below ):

name
parameters
notes 
addValidationRules( newRules )
  • newRules: JS object containing the new validation rule(s) to add ( see example below ).

The key / value pairs must be passed as:

  • key: the same used as data-subtype ( or type ) attribute, without dash and with Capital letter instead.
    Eg: to validate "mobile-number" ( as data-subtype ) the rule is "mobileNumber"
  • value: a function with these arguments:
    • string: the string the user put in the field
    • fiedlEl: the JS node element of the field
    It must return a JS object ( at least ) like: 
    {
    result: true
}
    In case of invalid field you can add the "errors" key to return the specific errors.
This will affect both old and new instances.
Can also be used to override existing validation rules. 
setOptions( options )
  • options: JS object with the fieldOptions/formOptions to override ( see Options section )
This will affect ONLY new instances

Examples of usage: 

Form.addValidationRules({
    myPassword: function( string, fieldEl ){
        let obj = {
            result: /^(?=.*[0-9])(?=.*[a-z])(?=.*[A-Z])[0-9a-zA-Z]{8,}$/.test(string)
        };
        if( !obj.result ){
            if( string.length < 8 ){ obj.errors.minlength = true; }
        }
        return obj;
    }
});

Form.setOptions( { fieldOptions: {maxFileSize: 100} } );
